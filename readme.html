Prueba técnica El propósito de esta prueba es evaluar al aplicante en los lenguajes PHP, JS, CSS, HTML y MySQL mediante el uso de CMS WordPress, utilizando un tema personalizado creado por el aplicante. Se evaluarán los siguientes puntos: 1. Creación de template Este debe de contener todos los archivos necesarios para que el theme funcione correctamente en WordPress. 2. Diseño del sitio El sitio debe replicar lo más fielmente posible el diseño compartido. El diseño debe utilizar variables globales en una hoja de estilo bien organizada. Se recomienda utilizar unidades vw y rem para asegurar que el texto se adapte adecuadamente a pantallas de escritorio. No se permite el uso de builders para la creación de los diseños. Diseño: https://www.figma.com/proto/c9JpRDbPUf1jdtMbqYejaN/Test?t=q6S8ORdKHYlUu P4u-1&scaling=min-zoom&content-scaling=fixed&page-id=0%3A1&node-id=1-15 3. Efectos/Animaciones Para todas las secciones se debe de aplicar un efecto de transición utilizando CSS y JS. Nota: Ver comentarios en Figma para más detalles de los efectos 4. Administración/Edición Los campos deben ser dinámicos, preferiblemente utilizando ACF para poder editar el contenido y luego ser llamarlos en el template, con el fin de facilitar al usuario final la edición del contenido sin tener que tocar el código. 5. Creación y almacenamiento de datos Al enviar el formulario presente en la página, los datos deben almacenarse en una tabla personalizada accesible desde el administrador de WordPress. Se debe implementar la capacidad de editar, eliminar y cambiar estados (pendiente/contactado/descartado). Por defecto, los datos deben almacenarse con el estado "pendiente". Nota: Esta funcionalidad se evaluará en cuanto al manejo de consultas MySQL en WordPress. Idealmente, esta funcionalidad debería ser implementada como un plugin que cree la tabla automáticamente en la base de datos si aún no existe.
